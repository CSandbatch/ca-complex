# docs/ — the demo surface

Screenshots and diagrams are part of the product, not afterthoughts. This repo
may be shown to someone with no background in the implementation, so these
assets carry real weight.

- `screenshots/` — PNGs of the shipped UI, generated by `scripts/screenshot.mjs`.
- `diagrams/` — `*.dot` source plus rendered `*.png`/`*.svg` (Graphviz):
  data flow, item types, lens specs, module architecture, graph bridge, place
  system, UI layout, state and interaction.
- `legacy/` — archived earlier READMEs. Reference only; not maintained.

## Local rules

- Diagrams are edited at the `.dot` source, then re-rendered. Do not hand-edit
  the rendered `.png`/`.svg`.
- Regenerate screenshots from the live UI after any material layout change.
- Update `README.md` and `../AGENTS.md` together with these assets when the
  visible structure changes. Prefer explanations a product viewer reads first and
  an engineer reads second.
- Do not delete a generated asset unless its replacement lands in the same pass.

## Gate

Visual: the asset matches the current shipped UI. After UI changes, confirm with
`npm run smoke` that the app still renders, then regenerate.

## Gotchas

- Screenshot filenames are referenced by `README.md`. Renaming one breaks the
  README image links.

Project doctrine: `../AGENTS.md`
